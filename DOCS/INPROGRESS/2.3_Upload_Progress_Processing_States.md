# Task 2.3: Upload Progress & Processing States

**Priority:** High
**Dependencies:** Task 2.2
**References:** PRD FR5, Task 2.2

## Description

Implement Fetch API with progress tracking to display upload percentage (0-100%). Show "Processing..." state after upload completes with a spinner or loading animation. Disable UI during processing. Trigger download when conversion completes and show success message.

## Subtasks

### 1. Implement Fetch API with Progress Tracking
**Acceptance Criteria:**
- [ ] Use `XMLHttpRequest` or Fetch with ReadableStream for upload progress.
- [ ] Display upload percentage (0-100%) in the UI.
- [ ] Ensure progress updates are smooth and responsive.

### 2. Implement Processing State
**Acceptance Criteria:**
- [ ] Show "Processing..." state after upload completes.
- [ ] Display a spinner or loading animation during processing.
- [ ] Disable UI elements during processing to prevent duplicate submissions.

### 3. Handle Response and Trigger Download
**Acceptance Criteria:**
- [ ] Trigger download automatically when conversion completes.
- [ ] Show a success message to the user.
- [ ] Ensure the UI is re-enabled after processing.

## Definition of Done

- [ ] Upload progress bar shows 0-100%.
- [ ] Processing state is clearly indicated.
- [ ] Download starts automatically on success.
- [ ] UI is disabled during processing.
